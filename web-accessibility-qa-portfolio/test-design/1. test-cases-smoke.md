# Test Cases - Smoke
**OpenCart - Header Keyboard Accessibility** 

## Overview
This document contains **Smoke test cases** for the *keyboard accessibility* on the **OpenCart** website.  
The focus is on verifying the keyboard accessibility of the header toolbar.

## Index
- [TC-SM-HeaderKeyboardAccessibility-01 – Keyboard navigation of header](#tc-sm-headerkeyboardaccessibility-01--keyboard-navigation-of-header)


## TC-SM-HeaderKeyboardAccessibility-01 – Keyboard navigation of header
**Type:** Smoke  
**Priority:** High  
**Related area:** Header toolbar

### Preconditions
- The local OpenCart Docker environment is running and accessible at http://localhost:8888.
- The home page is loaded and the header toolbar is visible
- It is possible to interact with the header toolbar using the keyboard

### Steps
1. Open http://localhost:8888/
2. Press Tab to reach the *Currency* element
3. Press Tab to reach the *Telephone* icon
4. Press Tab to reach the *My Account* element
5. Press Tab to reach the *Wish List* element
6. Press Tab to reach the *Shopping Cart* element
7. Press Tab to reach the *Checkout* element
8. Press Shift + Tab repeatedly until returning to the *Currency* element

### Expected Result
- The entire header is navigable via keyboard from left to right and vice versa
- Each interactive element is reachable using Tab and Shift + Tab
- Focus is visible on each interactive element during navigation
- The navigation order is logical and consistent
