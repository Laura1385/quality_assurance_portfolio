# Exploratory Test
**OpenCart – Inconsistent Keyboard Navigation Order**

## Overview
This document contains **exploratory test activities** for *keyboard accessibility* on the **OpenCart** website.   

The focus is on investigating focus visibility inconsistencies and navigation order issues on the home page, from the first header element to the first product actions in the *Featured* section, using keyboard-only navigation.

## TC-EXP-FeaturedProductKeyboardNavigation-01 – Investigate Focus Visibility Before Featured Product Actions
**Type:** Exploratory  
**Priority:** Medium  
**Related area:** Homepage carousel → Featured section  
**Timebox:** 13 minutes  
**Session type:** Focused exploratory session  
**Session goal:** Identify focus visibility inconsistencies and navigation order issues

### Objective
Investigate inconsistent keyboard focus visibility before reaching the *Featured* product actions.

### Test Charter
- Investigate keyboard focus visibility during homepage navigation
- Identify situations where focus visibility becomes unclear or temporarily disappears
- Observe keyboard navigation behavior near the homepage carousel/slideshow controls
- Verify whether the focus eventually reaches the *Featured* product action buttons
- Identify possible inconsistencies in navigation order or focus feedback

### Test Environment
- Browser: Google Chrome
- OS: macOS Tahoe 26.4.1
- Input device: Keyboard
- Observation date: 2026-05-21

### Exploration Areas
- Investigate keyboard navigation behavior on the home page
- Verify whether the cart icon of the first product in the *Featured* section is reachable using Tab navigation

### Observations
- Keyboard navigation remains logical and consistent before reaching the *Featured* section
- After the last element in the navigation category bar (*MP3 Players*), the visible focus indication becomes unclear
- Until reaching the *MP3 Players* element, a URL preview is displayed in the browser status bar in the bottom-left corner
- The visible focus appears again on the first product image in the *Featured* section after four Tab presses without visible focus indication
- Keyboard focus behavior near the first product action icons appears inconsistent across repeated navigation attempts
- In some cases, tooltip labels such as *Add to Cart* are displayed when the focus reaches the product action icons
- In other cases, no clear visual feedback is displayed after reaching the first product title in the *Featured* section
- Additional Tab presses may be required before visible feedback appears again

### Hypothesis
- The four Tab presses without clear visual focus indication may correspond to focusable carousel elements, such as the two carousel images and the two carousel indicators.
- These elements may receive keyboard focus without providing clearly perceivable visual feedback.

### Notes
- Browser DevTools inspection was attempted to identify the hidden focus target, but the focused element could not be clearly identified during navigation

### Outcome
- Potential keyboard focus visibility issue identified in the homepage carousel area
- Further investigation may be required to confirm whether all intermediate focusable elements provide sufficient visible focus indication

### Project Note
The observations reported in this session were collected on the public OpenCart demo website before the project was migrated to a local Docker environment for automation purposes.
