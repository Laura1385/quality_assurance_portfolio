# Exploratory Test
**Kate – Language Selection**

## Overview
This document contains **exploratory test activities** for the *Language Selection* feature in **Kate (KDE text editor)**.  
The focus is on investigating interaction-dependent UI behavior observed on first opening under varying conditions.

## Index
- [TC-EXP-DropdownScroll-01 – Language dropdown scroll behavior (Primary & Fallback)](#tc-exp-dropdownscroll-01--language-dropdown-scroll-behavior-primary--fallback)


## TC-EXP-DropdownScroll-01 – Language dropdown scroll behavior (Primary & Fallback)
**Type:** Exploratory  
**Priority:** Medium  
**Related area:** Settings → Language
**Timebox:** 23 minutes  
**Session type:** Focused exploratory session
**Session goal:** Identify interaction-dependent inconsistencies

### Objective
Investigate how the Primary and Fallback Language dropdown scroll behavior varies depending on initial user interaction and runtime conditions, especially on first opening.

### Test Charter
- Investigate dropdown scrolling behavior
- Identify conditions under which scrolling fails or works
- Observe impact of environment, timing, and input method

### Test Environment
- OS: Kubuntu 26.04 snapshot (Dec 2025)
- Execution: Virtual Machine (UTM)
- Display server: Wayland
- Input: Mouse, trackpad, keyboard
- Hardware constraints: Limited VM resources
- Observation date: 2026-01-28

### Exploration Areas
- Repeated opening and closing of the Configure Language dialog
- Different input methods (mouse wheel, trackpad, keyboard)
- Timing of interaction (immediate vs delayed interaction)
- Application restart vs same session
- VM performance state (fresh boot vs long-running session)
- Compare behavior between Primary and Fallback dropdowns
- Identify whether scrolling is enabled by interaction type or by dialog reinitialization (second opening)

### Observations
- The scroll issue occurs on the first opening of the dropdown (Primary and Fallback).
- On first opening, scrolling is disabled and navigation is limited to visible entries.
- Closing and reopening the Configure Language dialog enables scrolling.
- The issue reproduces after each application restart on the first opening.
- The behavior is independent of selection interaction; the second dialog opening is the effective trigger.

### Hypothesis
The scrolling instability may be related to focus handling or input event processing when the dropdown is first rendered.

### Notes
- The issue is reproducible on first opening and is mitigated by closing and reopening the dialog (second opening).
- Further validation is needed to determine whether this is focus initialization, event handling, or rendering timing.

- A defect was reported and documented in Jira: [SCRUM-3](https://lauraproto.atlassian.net/browse/SCRUM-3)
