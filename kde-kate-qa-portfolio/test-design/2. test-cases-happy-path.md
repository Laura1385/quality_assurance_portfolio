# Test Cases – Happy Path  
**Kate – Language Selection**
(Can the user complete the main flow correctly?) - Funziona

## Overview
This document contains **Happy Path test cases** for the *Language Selection* feature in **Kate (KDE text editor)**.  
The focus is on verifying that a user can successfully change the UI language and that the change behaves as expected across sessions.

## TC-HAPPY-01 – Change UI language (dropdown scroll)
**Type:** Happy Path  
**Priority:** High  
**Related area:** Settings → Language

### Preconditions
- Kubuntu VM is running and responsive
- Kate is installed and launched
- The Language settings dialog is accessible

### Steps (User)
1. Launch Kate
2. Open **Settings → Configure Language…**
3. Open the **Primary Language** dropdown
4. If the list is longer than the visible area, attempt to **scroll** inside the dropdown
   - Use mouse wheel / trackpad scroll
   - (Optional) Use keyboard navigation (Down arrow / PageDown)
5. Select a language that is **not currently selected** (e.g., Danish)
6. Click **OK** to confirm the language change

### Expected Result (What system does)
- The dropdown list **scrolls correctly** when languages exceed the visible area.
- The user can **reach and select any language** (including those not initially visible).
- After selecting a new language:
  - An informational message is displayed indicating that the language change will take effect after restarting the application (**message content is informational and non-blocking**).
  - The selected language is saved.
- The UI language is applied **after restart**
- No UI inconsistencies occur.

## TC-HAPPY-04 – Add Fallback Language and persist configuration
**Type:** Happy Path  
**Priority:** High
**Related area:** Settings → Language

### Preconditions
- Kubuntu VM is running and responsive
- Kate is installed and launched
- The Language settings dialog is accessible

### Steps (User)
1. Launch Kate
2. Open **Settings → Configure Language…**
3. Click on **Add Fallback Language**
4. Open the **Fallback Language** dropdown
5. Select a language different from the Primary language (e.g., Italian)
6. Click **OK** to confirm
7. Restart Kate

### Expected Result (What system does)
- A Fallback Language dropdown appears after clicking **Add Fallback Language**.
- The selected Fallback Language is displayed correctly.
- The configuration is saved after clicking **OK**.
- After restart:
  - The Fallback Language remains configured.
  - The **Remove** button is visible.
  - The **Add Fallback Language** button remains available.
- No UI inconsistencies occur.

## TC-HAPPY-05 – Remove Fallback Language and persist configuration
**Type:** Happy Path  
**Priority:** Medium  
**Related area:** Settings → Language

### Preconditions
- Kubuntu VM is running and responsive
- Kate is installed and launched
- The Language settings dialog is accessible
- A Fallback Language is already configured

### Steps (User)
1. Launch Kate
2. Open **Settings → Configure Language…**
3. Click on **Remove** button
4. Click **OK** to confirm
5. Restart Kate

### Expected Result (What system does)
- The configured Fallback Language is removed after clicking **Remove**.
- The configuration is saved after clicking **OK**.
- After restart:
  - No Fallback Language is configured.
  - The **Remove** button is no longer visible.
  - The **Add Fallback Language** button remains available.
- No UI inconsistencies occur.
